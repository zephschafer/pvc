## What Failed (all fixed during this run)

- **Template path resolution in copied TF work dirs**
  Terraform `path.module` resolves to `.` in the work directory, not the source module path. `templatefile()` calls failed.
  [→ Finding F-052: Major / Runtime — Fixed]

- **Airflow base image tag `2.9-python3.12` does not exist on Docker Hub**
  Short-form `major.minor-pythonX.Y` tags are not published by Apache Airflow.
  [→ Finding F-053: Major / Runtime — Fixed]

- **`pip install` as root blocked in Airflow 2.10+ images**
  The Airflow base image overrides pip to refuse installs as root.
  [→ Finding F-054: Major / Runtime — Fixed]

- **docker-compose `command: version` + `entrypoint:` interaction corrupts bash**
  `command:` value is passed as `$0` to `bash -c`, causing bash to parse flags as commands.
  [→ Finding F-055: Major / Runtime — Fixed]

- **Port 8080 hard-coded for Airflow webserver**
  Conflicts with other services (e.g., spark-iceberg). Needed configurable host port.
  [→ Finding F-056: Minor / UX — Fixed]

- **`DockerOperator` `volumes` parameter removed in providers-docker 3.x**
  DAG import error: `Invalid arguments were passed to DockerOperator: volumes`. Must use `mounts`.
  [→ Finding F-057: Major / Runtime — Fixed]

- **Airflow REST API auth backend not configured in docker-compose**
  All API calls returned 401 without `AIRFLOW__API__AUTH_BACKENDS` env var.
  [→ Finding F-058: Minor / UX — Fixed]
